# P9.2 — Offline-first PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a service worker + manifest upgrade + directory JSON API so members can browse the annuaire offline after first visit, with proper data-saver respect, per-user cache isolation, and a clean offline UX.

**Architecture:** Vanilla JS service worker served from `/sw.js` via a small Django view (root scope required), with cache-first for static assets, network-first for app-shell HTML, and stale-while-revalidate for the new `/api/annuaire.json` endpoint. Per-user cache scoping via a session-hash query parameter prevents cross-user data leaks on shared devices. Manifest upgraded from `minimal-ui` to `standalone` for true app-feel when installed.

**Tech Stack:** Django 5, vanilla JS (no Workbox), WhiteNoise (CompressedManifestStaticFilesStorage), pytest. No new Python dependencies.

**Spec:** `docs/superpowers/specs/2026-05-08-p9-2-pwa-design.md`.

## Locked design decisions for this plan

- **Service worker scope is root (`/`).** Therefore the SW file must be served from `/sw.js` (not `/static/js/sw.js`) — browsers limit scope to ≤ serving path. Implementation: a Django view reads the file from `STATICFILES_DIRS` at request time and serves it with `Service-Worker-Allowed: /` and `Cache-Control: no-cache, max-age=0`.
- **Per-user JSON cache scoping** via a `?u=<hash>` query string. Hash is `sha256(user.id || username || SECRET_KEY[:32])[:16]`. Different users → different cache keys → no leak on device-sharing. Hash exposes neither user PII nor session tokens.
- **Cache strategies**:
  - `/static/*` → cache-first (WhiteNoise hashes change URLs on content change).
  - `/api/annuaire.json?u=…` → stale-while-revalidate.
  - App-shell paths (`/`, `/annuaire/`, `/aide/`, `/guide/`, `/profil/`, `/membres/<slug>/`) → network-first with cache fallback.
  - Everything else (POST, `/accounts/`, `/admin/`, `/gestion/`, cross-origin) → pass-through (no SW interference).
  - Cloudinary URLs → cross-origin, pass-through. Browser HTTP cache handles repeats.
- **SW versioning:** single constant `VERSION` at top of `sw.js`. Operator bumps it on release. Activate handler deletes any cache that doesn't end in `-${VERSION}`.
- **Install prompt gating:** localStorage counter `pwa_visit_count`; button shown only when `count >= 2`. JS captures `beforeinstallprompt`, defers, fires on button click.
- **Offline banner:** static element in `base.html`, hidden by default (`hidden` attr), toggled by inline JS that listens for `online`/`offline` events.
- **JSON endpoint serialization:** mirrors what `directory_view` would expose, respecting `member.show_*` flags AND viewer's `data_saver` (server emits `photo_url: ""` when data_saver is on).
- **Whitelist additions:** `/sw.js` and `/offline/` are in `LOGIN_REQUIRED_WHITELIST`. The JSON endpoint is NOT whitelisted (member-only).

```python
from pathlib import Path

from django.http import FileResponse, HttpResponseNotFound


def service_worker_view(_request):
    """Serve /sw.js with root scope. Read from STATICFILES_DIRS at request time;
    not via collectstatic because the URL must remain stable (no content hash)."""
    sw_path = Path(django_settings.BASE_DIR) / "static" / "js" / "sw.js"
    if not sw_path.exists():
        return HttpResponseNotFound("sw.js not found")
    response = FileResponse(open(sw_path, "rb"), content_type="application/javascript")
    response["Service-Worker-Allowed"] = "/"
    response["Cache-Control"] = "no-cache, max-age=0"
    return response
```

```javascript
// Service worker for Les Retrouvailles.
// Spec: docs/superpowers/specs/2026-05-08-p9-2-pwa-design.md.
//
// Cache buckets (key includes VERSION so bumping invalidates everything):
//   STATIC_CACHE — /static/* (cache-first)
//   SHELL_CACHE  — app-shell HTML routes (network-first w/ cache fallback)
//   API_CACHE    — /api/* (stale-while-revalidate)
//
// To force-update on release: bump VERSION below.

const VERSION = 'v1';
const STATIC_CACHE = `retrouvailles-static-${VERSION}`;
const SHELL_CACHE = `retrouvailles-shell-${VERSION}`;
const API_CACHE = `retrouvailles-api-${VERSION}`;

const APP_SHELL_PATHS = ['/', '/annuaire/', '/aide/', '/guide/', '/profil/', '/charte/'];
const OFFLINE_URL = '/offline/';

self.addEventListener('install', (event) => {
  self.skipWaiting();
  event.waitUntil(
    caches.open(SHELL_CACHE).then((cache) => cache.add(OFFLINE_URL))
  );
});

self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((names) =>
      Promise.all(
        names
          .filter((n) => n.startsWith('retrouvailles-') && !n.endsWith(`-${VERSION}`))
          .map((n) => caches.delete(n))
      )
    ).then(() => self.clients.claim())
  );
});

self.addEventListener('fetch', (event) => {
  const request = event.request;
  if (request.method !== 'GET') return;

  const url = new URL(request.url);
  if (url.origin !== self.location.origin) return;

  // Never intervene on auth, admin, gestion, or webhook surfaces.
  if (
    url.pathname.startsWith('/accounts/') ||
    url.pathname.startsWith('/admin/') ||
    url.pathname.startsWith('/gestion/') ||
    url.pathname.startsWith('/webhooks/')
  ) {
    return;
  }

  if (url.pathname.startsWith('/static/')) {
    event.respondWith(cacheFirst(request, STATIC_CACHE));
    return;
  }

  if (url.pathname.startsWith('/api/')) {
    event.respondWith(staleWhileRevalidate(request, API_CACHE));
    return;
  }

  const isAppShell =
    APP_SHELL_PATHS.includes(url.pathname) ||
    url.pathname.startsWith('/membres/');
  if (isAppShell) {
    event.respondWith(networkFirst(request, SHELL_CACHE));
    return;
  }

  // Everything else: pass-through.
});

async function cacheFirst(request, cacheName) {
  const cached = await caches.match(request);
  if (cached) return cached;
  try {
    const response = await fetch(request);
    if (response.ok) {
      const cache = await caches.open(cacheName);
      cache.put(request, response.clone());
    }
    return response;
  } catch (e) {
    return caches.match(OFFLINE_URL);
  }
}

async function networkFirst(request, cacheName) {
  try {
    const response = await fetch(request);
    if (response.ok) {
      const cache = await caches.open(cacheName);
      cache.put(request, response.clone());
    }
    return response;
  } catch (e) {
    const cached = await caches.match(request);
    return cached || (await caches.match(OFFLINE_URL));
  }
}

async function staleWhileRevalidate(request, cacheName) {
  const cache = await caches.open(cacheName);
  const cached = await cache.match(request);
  const fetchPromise = fetch(request)
    .then((response) => {
      if (response.ok) cache.put(request, response.clone());
      return response;
    })
    .catch(() => null);
  if (cached) return cached;
  const fresh = await fetchPromise;
  if (fresh) return fresh;
  return caches.match(OFFLINE_URL);
}
```

```python
def user_session_hash(user) -> str:
    """Stable 16-char hex digest used as a per-user query parameter on
    /api/annuaire.json so the service-worker cache key is distinct per
    user. Avoids serving user A's directory snapshot to user B on a
    shared device. Hash inputs (user.id + username + secret prefix) are
    not exposed in the URL — only the digest is.
    """
    secret = settings.SECRET_KEY[:32]
    raw = f"{user.id}:{user.username}:{secret}"
    return hashlib.sha256(raw.encode("utf-8")).hexdigest()[:16]
```

## Rollback procedure

If P9.2 ships and breaks something in prod:

1. Revert the merge commit: `git revert -m 1 <merge-sha>` and push.
2. The SW's `unregister` is not automatic on revert. Tell affected users to:
   - DevTools → Application → Service Workers → Unregister, OR
   - The next release with a bumped `VERSION` will activate, and the new SW (which won't be re-registered if the JS is gone) will eventually expire.
3. Alternatively, ship a "kill switch" `sw.js` that just calls `self.registration.unregister()` and `caches.keys().then((names) => Promise.all(names.map((n) => caches.delete(n))))` — restores all clients to no-SW state on next page load. Keep this two-line variant of `sw.js` ready in `docs/runbooks/pwa-kill-switch.md` if a real incident happens; we don't ship it pre-emptively.
